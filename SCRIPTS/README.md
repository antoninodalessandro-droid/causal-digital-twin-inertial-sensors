# Causal and uncertainty-aware digital-twin framework for ultra-low-noise geoscientific inertial sensors

This repository contains the Python scripts used to reproduce the numerical figure-generation workflow associated with the manuscript:

**D'Alessandro, A. (2026). _Causal and uncertainty-aware digital-twin framework for ultra-low-noise geoscientific inertial sensors_.**

The code implements deterministic, physics-based calculations for a causal and uncertainty-aware digital-twin framework for ultra-low-noise geoscientific inertial sensors. The scripts reproduce the frequency-domain transfer-function analyses, acceleration self-noise budgets, metrological bounds, parameter-sensitivity tests, crossover-frequency diagnostics, and design-regime maps presented in the manuscript.

## Repository content

The computational material included in this archive consists of standalone Python scripts for Figures 2-7 of the manuscript.

| Manuscript figure | Recommended script name | Description |
|---|---|---|
| Figure 2 | `scripts/Figure2_open_closed_loop_transfer.py` | Open-loop and closed-loop transfer functions for a representative ultra-long-period force-feedback seismometer digital twin. |
| Figure 3 | `scripts/Figure3_noise_budget.py` | Frequency-dependent acceleration self-noise budget, including Brownian, readout, actuator/feedback, quantization, clock, and total self-noise terms. |
| Figure 4 | `scripts/Figure4_metrological_bounds.py` | Comparison between fundamental thermal noise limits, ideal force-feedback bounds, implementation-dependent bounds, and achieved total self-noise. |
| Figure 5 | `scripts/Figure5_spectral_sensitivity.py` | Sensitivity of the total acceleration self-noise to effective ADC resolution (ENOB) and mechanical damping ratio. |
| Figure 6 | `scripts/Figure6_regimes_and_crossovers.py` | Noise-regime curves and crossover-frequency metrics as a function of effective ADC resolution. |
| Figure 7 | `scripts/Figure7_design_regime_map.py` | ENOB-frequency design-regime map and effective near-plateau bandwidth. |

Figures 1, 8, and 9 are conceptual graphical schematics prepared separately for the manuscript. They were not generated from executable software scripts and are therefore not included as part of the computational code archive.

## Software requirements

The scripts were written for Python 3 and use standard open-source scientific Python libraries:

- `numpy`
- `matplotlib`

A minimal `requirements.txt` file is therefore:

```text
numpy
matplotlib
```

No third-party source code is redistributed in this repository. The scripts depend only on the above Python packages and on the Python standard library.

## Installation

Clone the repository and install the required packages:

```bash
git clone https://github.com/YOUR-USERNAME/causal-digital-twin-inertial-sensors.git
cd causal-digital-twin-inertial-sensors
python -m venv .venv
```

Activate the environment.

On Linux/macOS:

```bash
source .venv/bin/activate
```

On Windows:

```bash
.venv\Scripts\activate
```

Then install the dependencies:

```bash
pip install -r requirements.txt
```

## Reproducing the figures

Each script is designed to be executable independently. From the repository root, run for example:

```bash
python scripts/Figure2_open_closed_loop_transfer.py
python scripts/Figure3_noise_budget.py
python scripts/Figure4_metrological_bounds.py
python scripts/Figure5_spectral_sensitivity.py
python scripts/Figure6_regimes_and_crossovers.py
python scripts/Figure7_design_regime_map.py
```

The generated figure files are saved by each script according to the output path specified internally. When preparing the final archived release, it is recommended to save all generated figures in an `outputs/` directory to keep the repository structure uniform.

## Data availability

No observational, laboratory, or field datasets are required to reproduce the figures generated by these scripts. The analyses are based on deterministic model calculations and representative parameter values explicitly defined within the scripts.

The repository therefore constitutes a software archive rather than an observational data archive. All numerical inputs required to reproduce Figures 2-7 are contained in the corresponding Python scripts.

## Methodological scope

The scripts are intended to reproduce the illustrative numerical analyses presented in the manuscript. They should not be interpreted as calibrated models of a specific commercial or experimental instrument. The adopted parameter values represent physically plausible design-stage configurations used to illustrate the causal digital-twin framework, noise-budget decomposition, and performance-metric definitions.

The main simplifying assumptions are:

- linear time-invariant sensor dynamics;
- single-degree-of-freedom mechanical representation;
- mutually incoherent noise sources;
- stationary noise spectra;
- independent parameter choices unless otherwise specified;
- absence of site-specific environmental noise and experimental calibration data.

These assumptions are consistent with the design-stage scope of the manuscript and delimit the interpretation of the predicted self-noise spectra and regime boundaries.
